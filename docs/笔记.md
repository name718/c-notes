# 文件读写与内存管理笔记

> 面向 C 语言基础学习，重点整理：文件读写、函数/方法区别、文本/二进制模式、动态内存申请与释放。

## 1. 函数、方法、文件指针的基础概念

| 概念 | 含义 | C 语言里怎么理解 | 常见例子 | 备注 |
| --- | --- | --- | --- | --- |
| 函数（function） | 完成某个功能的一段可复用代码 | C 语言核心调用单位 | `printf()`、`fopen()`、`malloc()` | C 里主要讲“函数” |
| 方法（method） | 依附于对象或类的函数 | 标准 C 没有类和对象概念 | 面向对象语言里常见 | 在 C 里通常不严格区分“方法” |
| 参数 | 调用函数时传进去的数据 | 决定函数怎么工作 | `fopen("a.txt", "r")` 中两个参数 | 形参在定义处，实参在调用处 |
| 返回值 | 函数执行后返回的结果 | 用来判断是否成功、得到结果 | `malloc()` 返回地址，`fopen()` 返回 `FILE *` | 要注意判空和判错 |
| 文件指针 | 指向文件控制块的指针 | 用来操作文件 | `FILE *fp;` | 几乎所有文件函数都要用它 |
| 缓冲区 | 临时存放数据的内存区域 | 提高读写效率 | `char buf[100];` | 文件读写时很常见 |

## 2. 类型占用字节数（表格）

> 说明：C 类型大小和平台/编译器有关，下面是 64 位环境最常见情况。面试里建议补一句“以 `sizeof` 实测为准”。  
> LP64（Linux/macOS 常见）：`long` 为 8 字节；LLP64（Windows 64 位常见）：`long` 为 4 字节。

| 类型 | LP64 常见字节数 | LLP64 常见字节数 | 说明 |
| --- | --- | --- | --- |
| `char` | 1 | 1 | C 标准保证 `sizeof(char) == 1` |
| `short` / `short int` | 2 | 2 | 至少不小于 `char` |
| `int` | 4 | 4 | 常见为 4 |
| `long` / `long int` | 8 | 4 | 平台差异最常考 |
| `long long` | 8 | 8 | 一般为 8 |
| `float` | 4 | 4 | IEEE 754 单精度常见 |
| `double` | 8 | 8 | IEEE 754 双精度常见 |
| `long double` | 16（常见） | 8/16（实现相关） | 差异较大，面试可说明“实现定义” |
| `_Bool` / `bool` | 1（常见） | 1（常见） | 来自 `<stdbool.h>` 的 `bool` 本质是 `_Bool` |
| 指针（如 `int *`） | 8 | 8 | 64 位进程下通常 8 字节 |

### 快速确认写法

```c
printf("sizeof(int) = %zu\n", sizeof(int));
printf("sizeof(long) = %zu\n", sizeof(long));
printf("sizeof(void*) = %zu\n", sizeof(void*));
```

## 3. 常见占位符（`printf` / `scanf`）

| 占位符 | 常见对应类型 | `printf` 用法 | `scanf` 用法 | 备注 |
| --- | --- | --- | --- | --- |
| `%d` | `int` | 输出有符号十进制整数 | 读入 `int` | 最常用整型 |
| `%u` | `unsigned int` | 输出无符号十进制整数 | 读入 `unsigned int` | 只表示非负值 |
| `%hd` / `%hu` | `short` / `unsigned short` | 输出短整型 | 读入短整型 | `scanf` 读 `short` 要加 `h` |
| `%ld` / `%lu` | `long` / `unsigned long` | 输出长整型 | 读入长整型 | `long` 在不同平台字节数可能不同 |
| `%lld` / `%llu` | `long long` / `unsigned long long` | 输出长长整型 | 读入长长整型 | 64 位整数常用 |
| `%c` | `char` | 输出单个字符 | 读入单个字符 | `scanf(\"%c\")` 会读取空白字符 |
| `%s` | `char *`（字符串） | 输出字符串 | 读入字符串 | `scanf(\"%s\")` 不自动防止越界 |
| `%f` | `float` / `double` | `printf` 中浮点实参会提升为 `double` | `scanf` 中 `%f` 对应 `float *` | 高频易错点 |
| `%lf` | `double` | `printf` 中可写 `%f` 或 `%lf`（效果通常一致） | `scanf` 中 `%lf` 对应 `double *` | `scanf` 里必须区分 `%f/%lf` |
| `%x` / `%X` | `unsigned int` | 输出十六进制（小写/大写） | 读入十六进制整数 | 调试位运算常用 |
| `%o` | `unsigned int` | 输出八进制 | 读入八进制整数 | 使用频率较低 |
| `%p` | 指针地址 | 输出地址 | - | 参数通常写 `(void *)ptr` |
| `%zu` | `size_t` | 输出 `sizeof` 结果 | 读入 `size_t`（`%zu`） | 处理长度/字节数推荐 |
| `%%` | 字面量 `%` | 输出一个 `%` | - | 例如 `printf(\"%%d\")` |

### 两个高频易错点

1. `scanf` 读浮点：`float` 用 `%f`，`double` 用 `%lf`。  
2. 打印 `sizeof(...)`：建议用 `%zu`，因为返回类型是 `size_t`。

## 4. 文件读写的典型流程

| 步骤 | 说明 | 示例思路 |
| --- | --- | --- |
| 1 | 定义文件指针 | `FILE *fp;` |
| 2 | 打开文件 | `fp = fopen("data.txt", "r");` |
| 3 | 判断是否打开成功 | `if (fp == NULL)` |
| 4 | 进行读取或写入 | `fgets()`、`fprintf()`、`fread()` 等 |
| 5 | 使用结束后关闭文件 | `fclose(fp);` |
| 6 | 必要时处理错误 | 输出提示、返回错误码 |

### 文本读取示例思路

```c
FILE *fp = fopen("data.txt", "r");
if (fp == NULL) {
    return 1;
}

char line[100];
while (fgets(line, sizeof(line), fp) != NULL) {
    printf("%s", line);
}

fclose(fp);
```

### 二进制写入示例思路

```c
FILE *fp = fopen("data.bin", "wb");
if (fp == NULL) {
    return 1;
}

int arr[3] = {10, 20, 30};
fwrite(arr, sizeof(int), 3, fp);

fclose(fp);
```

## 5. 常见文件操作函数总览

| 函数 | 作用 | 常见原型理解 | 成功时 | 失败时 | 常见用途 |
| --- | --- | --- | --- | --- | --- |
| `fopen()` | 打开文件 | 根据路径和模式打开文件 | 返回 `FILE *` | 返回 `NULL` | 读文件、写文件、追加 |
| `fclose()` | 关闭文件 | 结束文件使用并刷新缓冲区 | 返回 `0` | 返回 `EOF` | 文件结束后必须关闭 |
| `fgetc()` | 读一个字符 | 每次读取 1 个字符 | 返回字符 | 返回 `EOF` | 按字符读取 |
| `fputc()` | 写一个字符 | 每次写入 1 个字符 | 返回写入字符 | 返回 `EOF` | 按字符写入 |
| `fgets()` | 读一行字符串 | 最多读 `n-1` 个字符 | 返回目标数组指针 | 返回 `NULL` | 读一行文本 |
| `fputs()` | 写一个字符串 | 把字符串写入文件 | 非负值 | `EOF` | 写文本行 |
| `fprintf()` | 格式化写入文件 | 类似 `printf()` 但写到文件 | 返回写入字符数 | 负值 | 写结构化文本 |
| `fscanf()` | 格式化读取文件 | 类似 `scanf()` 但从文件读 | 返回成功匹配个数 | `EOF` 或较小值 | 读取格式化数据 |
| `fread()` | 按块读取二进制数据 | 按字节块读取 | 返回成功读取的项数 | 项数不足/0 | 读结构体、图片、音频等 |
| `fwrite()` | 按块写入二进制数据 | 按字节块写入 | 返回成功写入的项数 | 项数不足/0 | 写结构体、二进制文件 |
| `fseek()` | 移动文件位置指针 | 跳到指定偏移位置 | 返回 `0` | 非 `0` | 随机访问文件 |
| `ftell()` | 获取当前位置 | 返回当前偏移量 | 返回位置值 | 返回 `-1L` | 计算文件大小、调试 |
| `rewind()` | 回到文件开头 | 等价于移动到开头 | 无返回值 | - | 重新读取文件 |
| `remove()` | 删除文件 | 删除磁盘文件 | 返回 `0` | 非 `0` | 删除临时文件 |
| `rename()` | 重命名文件 | 修改文件名 | 返回 `0` | 非 `0` | 文件改名 |

## 6. `fopen()` 打开模式总结

| 模式 | 含义 | 文件不存在时 | 文件已存在时 | 是否可读 | 是否可写 | 典型场景 |
| --- | --- | --- | --- | --- | --- | --- |
| `r` | 只读打开文本文件 | 打开失败 | 从头读取 | 是 | 否 | 查看文本内容 |
| `w` | 只写打开文本文件 | 创建新文件 | 清空原内容再写 | 否 | 是 | 覆盖写入 |
| `a` | 追加写入文本文件 | 创建新文件 | 从末尾追加 | 否 | 是 | 日志追加 |
| `r+` | 读写打开文本文件 | 打开失败 | 从头开始读写 | 是 | 是 | 修改已有文本 |
| `w+` | 读写打开文本文件 | 创建新文件 | 清空原内容再读写 | 是 | 是 | 重建文件内容 |
| `a+` | 读写追加文本文件 | 创建新文件 | 末尾追加，可读 | 是 | 是 | 边读边追加 |
| `rb` | 只读打开二进制文件 | 打开失败 | 从头读取 | 是 | 否 | 读取图片/音频/结构体 |
| `wb` | 只写打开二进制文件 | 创建新文件 | 清空原内容再写 | 否 | 是 | 写二进制数据 |
| `ab` | 追加写入二进制文件 | 创建新文件 | 从末尾追加 | 否 | 是 | 追加二进制日志 |
| `rb+` / `r+b` | 读写打开二进制文件 | 打开失败 | 从头读写 | 是 | 是 | 修改二进制内容 |
| `wb+` / `w+b` | 读写打开二进制文件 | 创建新文件 | 清空原内容再读写 | 是 | 是 | 重写整个二进制文件 |
| `ab+` / `a+b` | 读写追加二进制文件 | 创建新文件 | 末尾追加，可读 | 是 | 是 | 边读边追加二进制数据 |

## 7. `w` 和 `wb`、`r` 和 `rb` 的区别

| 对比项 | 文本模式（如 `w`、`r`） | 二进制模式（如 `wb`、`rb`） |
| --- | --- | --- |
| 数据处理方式 | 按文本理解数据 | 按原始字节处理数据 |
| 换行处理 | 某些系统可能对 `\n` 做转换 | 不做换行转换 |
| 适合内容 | `.txt`、日志、配置文件 | 图片、音频、视频、结构体、序列化数据 |
| 是否关心“字符”含义 | 是 | 否 |
| 移植性考虑 | 文本更适合人读 | 二进制更适合程序原样保存 |

### 简单理解

- `w`：写**文本文件**，如果文件存在会先清空。
- `wb`：写**二进制文件**，如果文件存在也会先清空。
- `r`：读**文本文件**。
- `rb`：读**二进制文件**。
- 学图片、音频、结构体存盘时，优先想到 `rb` / `wb`。

## 8. 文本读写和二进制读写的常见搭配

| 场景 | 推荐函数 | 推荐模式 | 说明 |
| --- | --- | --- | --- |
| 一行一行读取文本 | `fgets()` | `r` | 适合配置文件、名单、日志 |
| 格式化写文本 | `fprintf()` | `w` / `a` | 适合生成可读报表 |
| 一个字符一个字符处理 | `fgetc()` / `fputc()` | `r` / `w` | 适合简单练习 |
| 按结构读取二进制数据 | `fread()` | `rb` | 适合结构体、数组 |
| 按结构写入二进制数据 | `fwrite()` | `wb` | 适合保存程序内部数据 |
| 文件末尾持续追加 | `fprintf()` / `fwrite()` | `a` / `ab` | 适合日志、记录流 |

## 9. 内存的几个核心概念

| 概念 | 含义 | 特点 | 常见位置 |
| --- | --- | --- | --- |
| 局部变量 | 在函数内部定义的变量 | 自动分配、自动释放 | 栈（stack） |
| 全局变量 | 函数外定义的变量 | 程序整个运行期间存在 | 静态存储区 |
| 静态变量 | `static` 修饰的变量 | 生命周期长，但作用域有限制 | 静态存储区 |
| 动态内存 | 程序运行时手动申请的内存 | 灵活，但要手动释放 | 堆（heap） |
| 野指针 | 指向非法地址的指针 | 很危险，容易崩溃 | 常见于误操作 |
| 空指针 | 不指向有效对象的指针 | 值为 `NULL` | 用于初始化、判空 |

## 10. 栈内存和堆内存的区别

| 对比项 | 栈（stack） | 堆（heap） |
| --- | --- | --- |
| 分配方式 | 编译器自动分配 | 程序员手动申请 |
| 释放方式 | 函数结束自动释放 | 必须手动 `free()` |
| 速度 | 一般较快 | 相对较慢 |
| 大小 | 通常较小 | 通常更灵活 |
| 生命周期 | 随作用域结束 | 直到手动释放 |
| 风险 | 返回局部变量地址会出错 | 容易内存泄漏、重复释放 |
| 适合场景 | 临时变量、小数据 | 动态数组、大对象、运行期大小不确定的数据 |

## 11. 申请内存的函数总结

| 函数 | 作用 | 原型理解 | 内存初值 | 使用场景 |
| --- | --- | --- | --- | --- |
| `malloc(size)` | 申请指定字节数内存 | 申请一块连续空间 | 未初始化，值不确定 | 最常用 |
| `calloc(n, size)` | 申请 `n` 个同类型空间 | 总字节数为 `n * size` | 自动清零 | 申请数组更方便 |
| `realloc(ptr, new_size)` | 调整已有内存大小 | 扩容或缩容 | 新增部分未初始化 | 动态扩展数组 |
| `free(ptr)` | 释放动态内存 | 归还堆空间 | - | 与前三者配套使用 |

## 12. `malloc`、`calloc`、`realloc`、`free` 的区别

| 对比项 | `malloc()` | `calloc()` | `realloc()` | `free()` |
| --- | --- | --- | --- | --- |
| 主要作用 | 申请内存 | 申请并清零 | 调整内存大小 | 释放内存 |
| 参数形式 | 总字节数 | 元素个数 + 单个大小 | 原指针 + 新大小 | 要释放的指针 |
| 返回值 | 新内存首地址 | 新内存首地址 | 可能是新地址 | 无返回值 |
| 初始内容 | 随机值/脏数据 | 全部置 0 | 保留旧数据的一部分 | 不适用 |
| 是否需要判空 | 需要 | 需要 | 需要 | 通常先判空再释放也可以 |

### 示例理解

```c
int *p1 = (int *)malloc(5 * sizeof(int));
int *p2 = (int *)calloc(5, sizeof(int));
p1 = (int *)realloc(p1, 10 * sizeof(int));
free(p1);
free(p2);
```

## 13. 常见内存问题总结

| 问题 | 含义 | 常见原因 | 后果 | 避免方法 |
| --- | --- | --- | --- | --- |
| 内存泄漏 | 申请后没有释放 | 忘记 `free()` | 程序占用内存越来越多 | 申请和释放成对出现 |
| 重复释放 | 同一块内存释放两次 | 多处 `free(ptr)` | 程序崩溃 | 释放后置 `NULL` |
| 野指针 | 指针指向无效地址 | 未初始化、释放后继续使用 | 崩溃、数据破坏 | 初始化为 `NULL` |
| 越界访问 | 访问超出内存范围 | 数组下标错误 | 未定义行为 | 严格检查长度 |
| 悬空指针 | 指针指向已释放内存 | `free()` 后未置空 | 再访问会出错 | `free(ptr); ptr = NULL;` |

## 14. 指针类型总结（全）

### 1）按“指向对象类型”分类

| 指针类型 | 示例声明 | 含义 | 常见用途 |
| --- | --- | --- | --- |
| `char *` | `char *p;` | 指向 `char` | 字符、字符串遍历 |
| `int *` | `int *p;` | 指向 `int` | 整数变量地址传参 |
| `float *` / `double *` | `double *p;` | 指向浮点类型 | 数值计算、数组处理 |
| `struct *` | `struct Student *p;` | 指向结构体对象 | 结构体传参、链表节点 |
| `union *` | `union Data *p;` | 指向共用体对象 | 共享内存数据访问 |
| `enum *` | `enum Color *p;` | 指向枚举变量 | 状态值传递 |
| `void *` | `void *p;` | 通用指针（无具体类型） | 泛型接口、内存函数参数 |
| 函数指针 | `int (*fp)(int, int);` | 指向函数 | 回调函数、策略分发 |

### 2）按“指针层级”分类

| 层级 | 示例 | 含义 | 典型场景 |
| --- | --- | --- | --- |
| 一级指针 | `int *p;` | 存放对象地址 | 访问变量、数组首元素 |
| 二级指针 | `int **pp;` | 存放一级指针地址 | 在函数中修改指针本身、`argv` |
| 多级指针 | `char ***ppp;` | 更高层地址间接访问 | 复杂数据结构、接口封装 |

### 3）最易混淆：`const` 和指针

| 声明 | 谁不能改 | 谁能改 |
| --- | --- | --- |
| `const int *p;` / `int const *p;` | 不能通过 `p` 改“值” | `p` 可改指向 |
| `int *const p = &a;` | `p` 不能改指向 | 可通过 `p` 改“值” |
| `const int *const p = &a;` | 值不能改 | 指向也不能改 |

### 4）数组相关的两类指针（高频考点）

| 写法 | 名称 | 含义 |
| --- | --- | --- |
| `int *arr[10];` | 指针数组 | `arr` 是数组，元素类型是 `int *` |
| `int (*p)[10];` | 数组指针 | `p` 是指针，指向“含 10 个 `int` 的数组” |

### 5）函数指针与函数指针数组

| 写法 | 含义 | 示例用途 |
| --- | --- | --- |
| `int (*fp)(int, int);` | `fp` 指向返回 `int`、参数为两个 `int` 的函数 | 选择不同计算函数 |
| `int (*ops[4])(int, int);` | `ops` 是函数指针数组 | 菜单/命令分发表 |

### 6）特殊与扩展修饰（了解即可）

| 写法 | 说明 |
| --- | --- |
| `volatile int *p;` | 指向易变对象（对象值可能被外部改变） |
| `int *volatile p;` | 指针变量本身易变 |
| `const volatile int *p;` | 只读且易变对象 |
| `int *restrict p;` | C99，承诺该指针是访问对象的唯一初始方式（利于优化） |

### 7）指针状态分类（面试常问）

| 状态 | 含义 | 风险 |
| --- | --- | --- |
| 空指针（`NULL`） | 不指向有效对象 | 解引用会崩溃 |
| 野指针 | 未初始化或非法地址 | 未定义行为 |
| 悬空指针 | 指向已释放内存 | 易崩溃/数据破坏 |

### 8）一条实用结论

- 记声明优先级时，先看变量名，再看它和 `[]`、`()`、`*` 的结合关系。  
- 背诵顺口版：`[]` 和 `()` 优先于 `*`，需要“谁先结合”就用括号。
## 15. 结构体和共用体的区别（含字节数计算）

| 对比项 | `struct`（结构体） | `union`（共用体/联合体） |
| --- | --- | --- |
| 内存分配方式 | 各成员分别占用内存 | 所有成员共享同一块内存 |
| 成员是否可同时有效 | 可以 | 通常只可靠使用“最后一次写入”的成员 |
| 大小计算思路 | 按成员顺序排布 + 对齐 + 末尾补齐 | 取最大成员大小 + 对齐 |
| 典型用途 | 描述一个对象的多个属性 | 节省内存、同一段数据的多种解释方式 |

### 字节数计算规则（默认对齐）

1. 每个成员的起始地址要满足该成员的对齐要求，不满足就插入填充字节。  
2. `struct` 总大小：成员排布完成后，再补齐到“最大成员对齐值”的整数倍。  
3. `union` 总大小：先取最大成员的 `sizeof`，再补齐到“最大成员对齐值”的整数倍。

### 可背公式

- `sizeof(struct) = align_up(成员排布后的总长度, max_align)`
- `sizeof(union) = align_up(max(各成员sizeof), max_align)`

### 例子（常见 64 位环境）

```c
struct S {
    char  c;   // 1
    int   i;   // 4
    short s;   // 2
};             // sizeof(S) 常见为 12（有填充）

union U {
    char  c;   // 1
    int   i;   // 4
    short s;   // 2
};             // sizeof(U) 常见为 4
```

## 16. 文件操作和内存操作的联系

| 主题 | 文件操作 | 内存操作 |
| --- | --- | --- |
| 操作对象 | 磁盘中的文件 | 运行中的内存空间 |
| 速度 | 相对较慢 | 相对较快 |
| 生命周期 | 文件可长期保存 | 程序结束后通常失效 |
| 常见函数 | `fopen()`、`fread()`、`fwrite()` | `malloc()`、`calloc()`、`realloc()`、`free()` |
| 常见配合 | 先读文件到内存，再处理数据 | 处理后再写回文件 |

## 17. 学习时最容易混淆的点

| 易混点 | 正确理解 |
| --- | --- |
| “函数”和“方法”一样吗？ | 不完全一样。C 语言里主要是函数，方法一般是面向对象语言里的说法。 |
| `w` 和 `a` 的区别 | `w` 会清空原文件重新写，`a` 是在文件末尾追加。 |
| `w` 和 `wb` 的区别 | 一个偏文本，一个偏二进制；处理数据方式不同。 |
| `malloc()` 和 `calloc()` 的区别 | `malloc()` 不清零，`calloc()` 会清零。 |
| `free()` 后指针还能用吗？ | 不能直接再用，最好立刻设为 `NULL`。 |
| 局部数组和 `malloc()` 数组区别 | 局部数组在栈上自动释放，`malloc()` 数组在堆上必须手动释放。 |
